# 🚦 十字路口交通仿真系统 (OS 进程管理项目)

本项目是基于 Python 多线程和信号量机制实现的十字路口交通调度模拟。 核心解决了多方向车辆并发通行、红绿灯周期切换以及特种车辆（消防车、救护车、警车）优先通行的同步与互斥问题。

## ✨ 核心特性

- **多线程调度**：利用 `threading` 模拟交通灯与多方向车流的并发。
- **经典同步机制**：通过 `Lock`, `Condition`, `Semaphore` 解决资源竞争与排队问题。
- **特权优先级**：实现急救车、消防车、警车的无视红灯特权通行，不受 FIFO 限制。
- **双前端展示**：
  - 🖥️ **控制台模式**：纯字符 ASCII 实时渲染路口排队状态。
  - 🌐 **Web 模式**：基于 Gradio 的可视化操作面板与大屏监控视图。

## 🛠️ 快速运行指南

### 1. 建立虚拟环境

建议使用 Python 3.8 或以上版本。在项目根目录下打开终端，执行以下命令创建并激活虚拟环境：

**Windows:**

```
python -m venv .venv
.\.venv\Scripts\activate
```

**macOS/Linux:**

```
python3 -m venv .venv
source .venv/bin/activate
```

### 2. 安装依赖

激活虚拟环境后，安装项目所需的第三方库（主要为 Gradio 等前端展示库）：

```
pip install -r requirements.txt
```

### 3. 运行项目

直接通过统一入口脚本启动：

```
python run.py
```

运行后会出现交互式菜单：

1. **控制台模式**：在终端内以 ASCII 字符画形式实时渲染路口状态。
2. **Web 模式**：启动本地 Gradio 服务，提供完整的可视化参数调节与状态监控面板。
3. **后端验证模式**：无 UI 静默运行，用于纯逻辑测试和断言检查。

## 📁 核心文件说明

- `core.py`：后端核心逻辑（交通灯守护线程、车辆生成引擎、调度循环）。
- `console.py`：终端控制台 UI 及 ASCII 动态视图渲染逻辑。
- `web.py`：基于 Gradio 构建的前端网页交互界面。
- `run.py`：项目统一启动入口及命令行参数解析。
- `build_exe.bat` / `traffic_sim.spec`：PyInstaller 打包配置与脚本。

## 📦 关于打包exe

本项目提供了一个 `build_exe.bat` 脚本，用于将核心逻辑打包为独立 `.exe` 执行文件。

> **⚠️ 注意**：由于 PyInstaller 与 Gradio 静态资源捆绑的兼容性问题，打包生成的 exe 版本**禁用了 Web 模式**，仅支持运行控制台和验证模式。若需使用 Web 界面进行调试或展示，请务必直接通过源码运行 `run.py`。
